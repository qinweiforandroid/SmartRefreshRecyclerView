# PR2 设计思考：README 与接入体验整理

## 现象

- PR1 已经完成内部重构和接口收敛，但 README 仍停留在旧表达。
- 项目对外有多条接入路径，但缺少明确推荐顺序。

## 初步观察

- sample 其实已经能覆盖大部分典型场景。
- 当前最大问题不是“没有示例”，而是“示例与 README 没有被串起来”。

## 问题拆解

1. README 是不是只要修版本号就够了？
2. 是否需要在 README 中直接给出推荐路径？
3. `LoadMoreResult` 是否应该进入 README 主文档？

## 当前判断

- 只修版本号远远不够。
- README 必须明确推荐路径。
- `LoadMoreResult` 应进入 README 主文档，而不是只停留在代码和设计文档里。

## 决策

- PR2 聚焦 README 与接入体验，不继续做内部重构。
- 先写 PR2 设计文档，再动 README。
- 用 sample 对照表降低理解成本。

## 取舍原因

- 当前最影响使用体验的是文档滞后，而不是内部结构。
- 如果不明确推荐路径，使用者会在多个实现之间来回试错。
- `LoadMoreResult` 已经是主接口方向，应同步到对外文档。

## 待确认

- README 中是否同时保留 `SmartV2RecyclerView` 的推荐说明，还是只作为兼容/特定场景入口描述

## 下一步

- 重写 README
- 明确模块说明、推荐路径、sample 映射和迁移说明
